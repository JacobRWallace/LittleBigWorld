## BULLET REBUILD

Original Bullet runtime:
/MTd

Needed:
/MDd

Opened:
x64 Native Tools Command Prompt for VS 2026

Commands:

cd C:\Users\jacob\Desktop\bullet3-3.25

mkdir build

cd build

cmake .. -G "Visual Studio 18 2026" -A x64 ^
-DCMAKE_POLICY_VERSION_MINIMUM=3.5 ^
-DBUILD_SHARED_LIBS=OFF ^
-DUSE_MSVC_RUNTIME_LIBRARY_DLL=ON

cmake --build . --config Debug

Reason:

* rebuild Bullet using /MDd runtime

## OUTPUT / INTERMEDIATE DIRECTORY CHANGE

->

* stores temporary compiler/linker files in centralized build folder
* keeps root project cleaner

Intermediate files:

* .obj
* .ilk
* .idb
* build logs

These are disposable temporary build artifacts.

Output Directory changed:

$(ShortProjectName)$(Platform)$(Configuration)\

->

$(SolutionDir)build$(Configuration)\

Reason:

* centralizes executable/runtime output
* avoids duplicate x64/Debug folders
* cleaner project structure

Output files:

* .exe
* .dll
* .pdb

These are the actual runnable application files.
